Cookie-Session-jwt-登录鉴权

发表于2023-06-19|更新于2023-06-19|业务逻辑
|字数总计:413|阅读时长:1分钟|阅读量:

Cookie

Cookie 是一种存储在客户端的小型文本文件，它可以通过浏览器向服务器发送请求时自动携带。在 Web 应用程序中，Cookie 通常用于存储用户的身份验证信息，比如用户 ID、用户名等。服务器可以使用 Cookie 来识别用户并提供个性化的服务。

Cookie 的缺点是它的数据存储在客户端，容易被篡改或盗用，因此需要进行加密和签名等安全措施。

Session

Session 是一种在服务器端存储用户信息的机制，它可以记录用户的登录状态、购物车信息等。在用户登录后，服务器会创建一个唯一的 Session ID，并将该 ID 存储在 Cookie 中发送给客户端。客户端在后续的请求中会携带该 Session ID，服务器可以通过该 ID 来查找用户的会话信息。

Session 的优点是它的数据存储在服务器端，相对比较安全，但是需要占用服务器的资源。

JWT

JWT（JSON Web Token）是一种轻量级的身份验证和授权机制，它可以在客户端和服务器之间传递信息。JWT 由三部分组成：头部、载荷和签名。头部包含了加密算法和类型等信息，载荷包含了用户的身份信息，签名用于验证数据的完整性。

JWT 的优点是它不需要在服务器端存储任何信息，可以在分布式系统中使用，但是需要注意保护好密钥，以防止被攻击者篡改数据。
